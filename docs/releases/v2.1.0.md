This release turns the repository into a clearer, more usable optimizer project.

It makes the supported path obvious, keeps the original implementation available for provenance, promotes the stopping and diagnostics workflow into the main supported codebase, and adds a carefully labeled SDS-inspired candidate branch that is real enough to test but not oversold.

## Why this release matters

- New users now have one clear starting point: `TopologicalAdamV2`.
- Older work is still preserved, but V1 and V2 are no longer tangled together.
- The internal coupling signal `J_t` is documented as a practical diagnostic rather than a grand claim.
- The SDS-inspired branch is included as a serious experiment, with honest benchmark language and no forced promotion.

## Included in v2.1.0

### 1. A clear supported path

- `TopologicalAdamV2` is the recommended optimizer.
- `TopologicalAdam` remains available as the legacy path.
- `TopologicalAdamSDS` is included as an experimental branch.

### 2. Practical diagnostics and stopping

- reusable field and coupling diagnostics
- reconnection-style stopping helper moved into supported code
- clearer examples for quickstart and stopping workflows

### 3. Cleaner repo structure

- explicit `v1.py`, `v2.py`, and `sds.py` modules
- shared diagnostics in `analysis.py`
- benchmark harness in `benchmarks.py`
- clearer docs, results, and MHD-connection pages

### 4. More honest benchmark framing

- the repository now distinguishes implementation facts from empirical findings
- `J_t` is no longer presented as uniquely diagnostic of the topological correction
- the SDS branch is treated as promising experimental work, not a default replacement

## Honest scope

This release does not claim that Topological Adam is already a benchmark-winning optimizer family.

What it does claim:
- the V2 path is now clean, supported, and usable
- the diagnostics workflow is real and reproducible
- the stopping helper is practical enough to include
- the SDS candidate branch is stable enough to test further

What remains open:
- whether the topological correction gives a durable optimization advantage on stronger workloads
- whether `J_t` transfers as a useful stopping or monitoring signal beyond the current synthetic setups
- whether the SDS branch earns promotion beyond experimental status
