v2.2.0 prepares Topological Adam as a clearer research-software package. It
makes `TopologicalAdamV3` the recommended optimizer path, includes the
experimental `TopologicalAdamV4` trajectory-topology branch, adds JOSS paper
materials, and removes stale research notes and one-off scripts from the
repository surface.

## Why this release matters

- New users now have one clear starting point: `TopologicalAdamV3`.
- V3 has explicit Adam/AdamW reduction tests and benchmarked behavior under
  both deterministic and noisy settings.
- V4 makes the trajectory-topology idea operational through projected
  turning/winding diagnostics and optional exact Vietoris-Rips H1 persistence.
- The repository now presents as an installable Python optimizer package, not
  a collection of internal drafts.
- JOSS materials, contribution guidelines, CI, packaging, citation metadata,
  and benchmark outputs are included in the repository.

## Included in v2.2.0

### 1. Recommended V3 optimizer path

- `TopologicalAdamV3` is the recommended optimizer.
- V3 uses slow and fast gradient-EMA fields, a coherence gate, and optional
  cautious updates.
- With `w_topo=0, cautious=False`, V3 reduces exactly to Adam or AdamW.
- The V3 audit and benchmark details are documented in `docs/v3-audit.md`.

### 2. Experimental V4 trajectory-topology branch

- `TopologicalAdamV4` computes topological information from the optimizer's
  own recent update trajectory.
- The default detector uses projected turning/winding statistics.
- Optional exact Vietoris-Rips H1 persistence is available through
  `persistence_every`.
- With `loop_gate=False`, V4 reduces exactly to Adam or AdamW.
- V4 is documented as specialized and experimental, not a general Adam
  replacement.

### 3. Benchmark and evidence cleanup

- A reviewer-facing real-data training benchmark is included in
  `examples/reference_training_benchmark.py`, with stored output in
  `reference_training_results.json`.
- V3 benchmark outputs are stored in `benchmark_v3_results.json`.
- Fresh-seed V3 confirmation is stored in `fresh_seed_confirmation.json`.
- V4 benchmark outputs are stored in `benchmark_v4_results.json`.
- The results documentation reports wins, parity cases, and losses.
- Synthetic and real-data tasks are labeled separately.

### 4. JOSS submission materials

- `paper.md` and `paper.bib` are included with the software.
- `docs/joss-readiness.md` maps repository files to JOSS review expectations.
- `CITATION.cff`, `CONTRIBUTING.md`, and the MIT license are present.
- The paper includes the required AI usage disclosure.

### 5. Packaging and CI

- `pyproject.toml` is the package metadata source of truth.
- The package exports V3, V4, and persistence helpers from
  `topological_adam.__init__`.
- GitHub Actions runs tests on Python 3.10 and 3.12.
- The PyPI workflow builds and checks distributions on pull requests and main
  pushes, and publishes only on a GitHub Release or an explicit manual publish
  dispatch.

### 6. Repository cleanup

- Stale internal memos and obsolete experiment entry points were removed.
- The README was rewritten around installation, usage, optimizer variants,
  reproducibility, tests, and JOSS materials.
- The remaining top-level files are package, paper, benchmark, documentation,
  and project-governance files.

## Honest scope

This release does not claim that Topological Adam is a universal replacement
for Adam.

What it does claim:

- V3 is the current recommended path for users who want to test the package.
- V3 is benchmarked, documented, and tested against exact Adam/AdamW reduction.
- V4 implements real trajectory-topology diagnostics and uses them
  operationally, while remaining experimental.
- The package is now structured for installation, review, testing, and
  further research.

What remains open:

- Broader optimizer benchmarks with fixed hyperparameter-search budgets.
- More real research workflows using the package outside the included
  benchmark scripts.
- Independent user feedback, issues, or pull requests.
- A tagged release archive and DOI after the release is accepted for JOSS or
  prepared for long-term citation.
